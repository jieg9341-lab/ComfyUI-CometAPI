# ComfyUI-CometAPI

作者：B 站「那颗星星188」

ComfyUI-CometAPI 是一套面向 ComfyUI 的多模态 API 创作节点系统。

它不是简单把几个模型接口塞进 ComfyUI，而是围绕真实创作流程，把 API 模型、参考素材、提示词、批量任务、结果卡片、局部运行、设置中心和内置对话助手整合到一套更像“无限画布创作台”的工作流里。

如果你经常在 ComfyUI 里接各种 API 生图、生视频、生音乐，CometAPI 的目标就是减少重复配置、减少混乱连线、减少上游重跑，让 API 创作更接近一个可长期迭代的画布系统。

## 为什么做 CometAPI

在 ComfyUI 中使用 API 模型时，经常会遇到这些问题：

- 多张参考图连接到同一个节点时，画布很快变成“连线地狱”。
- 生成结果想复用，但一接到下游就容易把上游整条链重新跑一遍。
- 不同 API 平台、不同模型的比例、分辨率、时长、参考素材数量都不一样。
- 批量生图很难稳定管理，失败、编号、输出目录、并发都容易混乱。
- 想用 LLM 写提示词，再接图像或视频，常常要复制粘贴。
- 图像、视频、音频、文本素材缺少统一的画布化管理方式。
- 只想重跑局部节点时，默认队列经常带着不相关上游一起跑。

CometAPI 主要解决的就是这些问题。

它把图像、视频、音乐、文本、批量、素材卡片、设置中心和对话助手统一在一套 ComfyUI 工作流中，让用户可以更自然地在画布里组织 API 创作过程。

## 核心能力概览

### 1. 统一 API 节点

CometAPI 现在主要包含这些节点：

- `Comet 图像`
- `Comet 批量图像`
- `Comet 异步图像提交`
- `Comet 异步图片收取`
- `Comet 文本`
- `Comet 视频`
- `Comet 异步视频提交`
- `Comet 异步视频收取`
- `Comet 音乐`
- `Comet 图像卡片`
- `Comet 视频卡片`
- `Comet 音频卡片`
- `Comet 文本卡片`
- `Comet 批量文本卡片`
- `Comet 文本追加`
- `CometAPI 设置中心`

设计思路是：用户不用记每个平台的接口细节，只需要按“我要生成什么”来选择节点。

图像、视频、音乐、文本都以统一节点入口呈现。节点数量相对克制，但覆盖的模型和场景更广。

### 2. 设置中心统一管理渠道

CometAPI 有独立设置中心，用于统一管理：

- 渠道启用状态。
- API URL。
- API Key。
- 模型列表。
- 模型别名。
- 单模型独立 Key。
- 文本模型接口格式。
- 自定义图像模型。
- 自定义 LLM 模型。
- 批量图像高级参数。
- 提示词预设。

这样用户不用在每个节点里重复填写 Key，也不需要在不同节点中维护重复模型配置。

当前内置方向包括：

- 图像：Grsai、RunningHub、ModelVerse、Apimart，以及自定义图像渠道。
- 视频：RunningHub、ModelVerse、Apimart，以及私有渠道扩展。
- 音乐：RunningHub Suno 系列。
- 文本：Grsai、RunningHub、ModelVerse、Apimart，以及自定义 LLM 渠道。

具体可用模型、参数和价格以对应 API 渠道实际支持为准。

### 3. 动态参数显示，减少踩坑

不同 API 模型支持的参数差异很大。

例如：

- 有的图像模型支持 4K，有的只支持 1K 或 2K。
- 有的视频模型支持文生，有的支持首尾帧，有的支持视频编辑。
- 有的模型可以接多张参考图，有的只能接一张。
- 有的模型支持视频、音频参考，有的只支持图片。

CometAPI 前端会根据当前选择的渠道、模型家族和模型，自动显示或隐藏相关参数。

例如：

- 选择图像模型时，显示适合该模型的比例、尺寸、质量等参数。
- 选择视频模型时，显示对应的视频模式、时长、分辨率、比例和素材限制。
- 选择音乐模型时，显示歌词模式、灵感模式、纯音乐模式、声线等参数。

这个细节看起来只是界面优化，但对实际使用非常重要。很多 API 报错不是模型坏了，而是用户传了模型不支持的参数。CometAPI 在前端和后端都做了约束，尽量减少这类问题。

### 4. 虚拟连线，解决多参考图混乱

ComfyUI 原生连线很强，但在多参考图工作流中很容易变乱。

CometAPI 做了虚拟连线机制。

你可以把多个图片输出拖到 Comet 节点的素材入口上。前端会记录这些连接，并在真正提交任务时，把它们编译成后端可以识别的真实输入，例如：

```text
image_1
image_2
image_3
...
```

对用户来说，画布更干净；对后端来说，仍然是标准 ComfyUI 输入。

这个机制需要兼容 ComfyUI 的校验、序列化、节点复制、工作流保存和任务提交，不只是视觉上画一个入口。

### 5. 素材卡片，让结果可以复用

Comet 图像卡片、视频卡片、音频卡片不是普通预览节点。

它们会把生成结果保存成素材引用，并把这个引用写回节点属性。

这样带来的好处是：当你后续复用这个卡片里的图片、视频或音频时，可以直接读取已经保存的素材，而不是每次都重新触发上游生成链。

这对 API 创作很重要，因为 API 生成通常有成本、有等待时间、有随机性。如果每次下游复用都重跑上游，不仅浪费钱，也会破坏已经选好的结果。

图像卡片还支持：

- 图像预览。
- 多图管理。
- 主图选择。
- 网格查看。
- `image` 输出。
- `batch_image` 输出。

批量图像出来后，如果下游没有接 `batch_image`，它会返回轻量占位，避免大批量图片直接构造 tensor 占用内存；如果下游确实需要，再按需构造。

### 6. 单点运行，减少整图重跑

CometAPI 支持局部运行，也就是节点上的“单点运行”。

在复杂工作流里，很多时候只想重跑某一个图像节点、某一个视频节点，或者某张卡片前面的局部链路。

CometAPI 后端注册了自己的局部运行路由，前端会收集目标节点的上游子图，只执行允许的安全节点集合。执行结果会写进 ComfyUI 历史记录，并同步回卡片 UI。

对用户来说，它就像在画布上多了一个“只跑这里”的按钮。

对 API 创作来说，它能明显减少等待、减少误触发、减少试错成本。

### 7. 真批量图像，不只是简单循环

`Comet 批量图像` 支持多种批量方式：

- 常规批量。
- 文件夹批量。
- 多提示词批量。
- 多参考图批量。
- 图片和提示词一一配对。
- 每张图跑全部提示词。
- 全部参考图作为一组。

它还提供更贴近真实生产的控制项：

- 批量并发数。
- 最大任务数。
- 文件夹预览最大张数。
- 失败策略。
- 连续失败熔断。
- 独立输出文件夹。
- 文件名带任务编号、提示词编号、参考图编号。

这个功能适合：

- 角色换装。
- 产品图批改。
- 分镜批量风格化。
- 参考图批量再创作。
- 多提示词批量测试。

### 8. 异步任务，适合长耗时生成

图片和视频 API 经常会遇到耗时较长、平台排队、需要轮询结果的情况。

CometAPI 提供异步节点：

- `Comet 异步图像提交`
- `Comet 异步图片收取`
- `Comet 异步视频提交`
- `Comet 异步视频收取`

基本流程是：

1. 提交节点负责创建远端任务。
2. 输出任务 ID。
3. 收取节点根据任务 ID 查询结果。
4. 结果完成后保存到本地输出目录。

这样可以把“提交任务”和“等待结果”拆开，更适合长视频、批量图片、远端异步平台等场景。

### 9. 提示词卡片、批量文本和画布内提示词组织

Comet 文本卡片和批量文本卡片用于解决提示词组织问题。

LLM 生成的文字可以直接进入文本卡片。批量文本卡片可以解析：

- 普通换行列表。
- 编号列表。
- JSON 数组。
- LLM 输出中的多条提示词。

这样可以搭建类似：

```text
LLM 生成批量提示词 -> 批量文本卡片 -> Comet 批量图像
```

的自动流程。

前端还支持提示词增强输入：

- 用 `@` 引用画布上的图片、视频等卡片素材。
- 用 `/` 调出内置提示词预设。
- 管理角色三视图、场景四视图、分镜图、影视故事板等模板。
- 用户可以在设置中心新增、编辑自己的预设。

这部分的价值是把“复制提示词模板”和“素材引用”变成画布内交互，而不是外部文档复制粘贴。

### 10. 视频和音乐工作流

Comet 视频节点会根据不同渠道和模型家族显示不同参数。

可能支持的模式包括：

- 文生。
- 图生。
- 首尾帧。
- 全能参考。
- 多图参考。
- 视频编辑。
- 运动控制。

支持素材类型可能包括：

- 图片。
- 视频。
- 音频。

具体取决于渠道和模型。

Comet 音乐节点主要用于音乐生成任务，当前方向包括 RunningHub Suno 系列。它可以配合音频卡片保存和复用结果。

### 11. 内置 Comet 对话模块

CometAPI 还集成了一套 Comet 对话模块。

它可以在 ComfyUI 内直接打开一个 AI 对话窗口，用于：

- 解释 ComfyUI 报错。
- 编写或优化提示词。
- 看代码。
- 处理 Markdown。
- 查看表格。
- 发送图片、PDF、音频、视频、文本等附件。
- 调用部分图片模型直接在对话中生图。

对话模块支持：

- 宽屏模式。
- 半屏模式。
- 多会话。
- Markdown 渲染。
- 代码块高亮。
- 表格显示。
- 复制和重新生成。
- 图片粘贴。
- 图片复制、下载、重新生成、多版本翻页。

在 CometAPI 集成版中，对话模块可以读取 CometAPI 设置中心里已经配置好的文本模型渠道，减少重复配置。

## 推荐使用流程

### 基础图像生成

1. 添加 `Comet 图像`。
2. 选择渠道和模型。
3. 输入提示词。
4. 选择比例和分辨率。
5. 输出接到 `Comet 图像卡片`。
6. 点击单点运行或运行工作流。

重点：图像卡片会保存结果，下游复用时不必每次重跑上游。

### 多参考图生成

1. 准备多张参考图或图像卡片。
2. 把它们拖到 Comet 节点素材入口。
3. 通过虚拟连线保持画布简洁。
4. 提交时自动编译成真实后端输入。

重点：视觉上简洁，后端仍然兼容标准 ComfyUI 输入。

### LLM 到图像

1. 用 `Comet 文本` 生成提示词。
2. 接到 `Comet 文本卡片` 或图像节点提示词输入。
3. 用 `/` 预设或 `@` 素材引用增强提示词。
4. 接入 `Comet 图像` 或 `Comet 批量图像`。

重点：把 LLM、提示词和图像生成串成画布内流程。

### 批量图像

1. 用 `Comet 批量文本卡片` 准备多条提示词。
2. 可选：准备参考图或参考图文件夹。
3. 添加 `Comet 批量图像`。
4. 设置并发、最大任务数、失败策略。
5. 运行后查看独立输出文件夹和结果摘要。

重点：批量不是简单循环，而是带编号、失败策略、输出管理和熔断控制。

### 视频和音乐

1. 添加 `Comet 视频` 或 `Comet 音乐`。
2. 选择渠道、模型和模式。
3. 根据模型能力连接图片、视频或音频参考。
4. 输出到视频卡片或音频卡片。

重点：模型切换时参数会动态变化，尽量只显示当前模型可用参数。

## 安装方式

进入 ComfyUI 的 `custom_nodes` 目录：

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/jieg9341-lab/ComfyUI-CometAPI.git
```

然后重启 ComfyUI。

如果你下载的是 zip 包，请解压后确保目录类似：

```text
ComfyUI/
└─ custom_nodes/
   └─ ComfyUI-CometAPI/
      ├─ __init__.py
      ├─ nodes.py
      ├─ cometapi_chat.py
      ├─ web/
      │  └─ comet_api.min.js
      └─ private_channels/
```

## 依赖说明

插件运行在 ComfyUI 环境中，默认依赖 ComfyUI 已有的 Python、PyTorch、Pillow 等基础环境。

如果缺少网络请求相关依赖，可以在 ComfyUI 的 Python 环境中安装：

```bash
python -m pip install requests aiohttp
```

## 私有渠道扩展

CometAPI 支持通过 `private_channels` 目录扩展私有渠道。

公开分发版不会内置具体私有渠道业务逻辑，只保留识别、加载和适配机制。

如需编写私有渠道，请阅读：

```text
private_channels/私有渠道适配指南.md
```

私有渠道通常由两个文件组成：

- `<channel_id>.json`
- `<channel_id>.py`

注意：

- 不要把 API Key 写进私有渠道文件。
- API Key 应在插件设置页填写。
- 不要把客户私有渠道逻辑提交到公开仓库。
- 修改私有渠道 Python 文件后通常需要重启 ComfyUI。

## 常见问题

### 这个插件自带 API Key 吗？

不自带。你需要使用自己的 API 渠道和 API Key。

### 这个插件绑定某个商业渠道吗？

不绑定。你可以使用自己想用的 API 渠道。不同渠道的模型、价格、稳定性、参数支持都以渠道实际情况为准。

### 为什么某个模型参数不生效？

不同渠道和模型支持的参数不同。比如比例、分辨率、参考图数量、视频时长、编辑模式等，不一定每个平台都支持。

### 为什么某些模式下参数会隐藏？

这是插件的动态参数机制。当前模型不需要或不支持的参数会被隐藏，减少误传参数导致的 API 报错。

### 为什么多参考图没有很多线？

这是虚拟连线机制。前端记录素材连接，提交时再编译成后端真实输入。

### 为什么卡片里的图片复用时不重跑上游？

卡片会保存素材引用。后续复用时可以直接读取已保存素材，而不是重新触发上游生成链。

### 图片或视频任务很慢怎么办？

远端生成任务耗时取决于平台排队、模型速度、分辨率、并发数量和网络情况。长任务建议使用异步节点。
