# ComfyUI-CometAPI

ComfyUI-CometAPI 是一个面向 ComfyUI 的多模态 API 节点插件，作者为 B 站「那颗星星188」。

它把文本、图像、视频、音乐等常见 AI API 调用整理成 ComfyUI 节点，并提供一套更适合在画布里操作的前端交互：统一渠道设置、模型选择、参考图输入、单点运行、异步任务、结果卡片、以及内置对话助手。

## 核心功能

- ComfyUI 内的统一 API 节点，支持文本、图像、视频、音乐等多种任务。
- 设置中心统一管理 API 渠道、API Key、模型列表和格式。
- 支持多个预置渠道，并支持用户通过私有渠道机制扩展自己的平台。
- 图像节点支持参考图输入、比例、分辨率、批量生成和结果返回。
- 视频节点支持文生、图生、首尾帧、多参考、视频编辑等不同模式，具体能力取决于渠道和模型。
- 音乐节点支持 RunningHub Suno 系列模型。
- 文本节点支持常见 LLM 接口，用于提示词、报错分析、文本处理等。
- 异步图片 / 异步视频节点支持提交任务和收取任务分离，适合长耗时任务。
- 图像、视频、音频、文本卡片节点可作为画布中的资源边界，便于保存、预览和复用结果。
- 内置 Comet 对话模块，可在 ComfyUI 内直接进行 AI 对话、看 Markdown、复制代码、发送附件、调用图片模型。
- 前端已打包为 `web/comet_api.min.js`，分发版无需安装前端构建依赖。

## 适合场景

- 在 ComfyUI 内直接调用外部图像、视频、音乐、文本 API。
- 把 API 生图、生视频结果接入现有 ComfyUI 工作流。
- 需要一套统一的渠道配置，而不是在每个节点里反复填写 API Key。
- 需要批量生成图片，并把每次结果保存在 ComfyUI 输出目录。
- 需要异步提交长任务，稍后再收取结果。
- 需要在 ComfyUI 页面里直接打开 AI 对话助手，处理报错、提示词和代码问题。
- 需要接入自有或第三方平台，但不想改插件主体代码。

## 安装方式

进入 ComfyUI 的 `custom_nodes` 目录：

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/jieg9341-lab/ComfyUI-CometAPI.git
```

然后重启 ComfyUI。

如果你是下载 zip 包，请解压后确保目录结构类似：

```text
ComfyUI/
└─ custom_nodes/
   └─ ComfyUI-CometAPI/
      ├─ __init__.py
      ├─ nodes.py
      ├─ cometapi_chat.py
      ├─ web/
      │  └─ comet_api.min.js
      └─ private_channels/
```

## 第一次使用

1. 安装插件并重启 ComfyUI。
2. 打开 ComfyUI 页面。
3. 在 CometAPI 设置中心填写渠道的 API URL、API Key 和模型配置。
4. 在节点列表中搜索 `Comet`。
5. 添加需要的节点，例如 `Comet 图像`、`Comet 视频`、`Comet 文本`。
6. 选择渠道、模型、比例、分辨率等参数。
7. 连接输入输出，运行工作流。

## 设置中心

CometAPI 的设置中心用于管理全局配置。

通常你需要配置：

- 渠道是否启用。
- 渠道 API URL。
- 渠道通用 API Key。
- 模型列表。
- 单模型独立 Key。
- 文本模型接口格式。
- 自定义渠道配置。

如果模型配置了独立 Key，会优先使用模型独立 Key；否则使用渠道通用 Key。

## 主要节点

### Comet 图像

用于调用图像生成或图像编辑类 API。

常见能力：

- 文生图。
- 图生图。
- 多参考图。
- 比例选择。
- 分辨率选择。
- 单点运行。
- 输出 IMAGE。

支持的具体参数取决于渠道和模型。不同模型对参考图数量、分辨率、比例的支持可能不同。

### Comet 批量图像

用于批量提交图像生成任务。

适合：

- 同一提示词批量生成多张。
- 文件夹参考图批处理。
- 多任务并发。
- 输出批量图片和任务信息。

### Comet 异步图像提交 / Comet 异步图片收取

适合耗时较长或平台异步返回的图片任务。

基本流程：

1. 使用 `Comet 异步图像提交` 提交任务。
2. 得到任务 ID。
3. 稍后使用 `Comet 异步图片收取` 收取结果。

这样可以避免长时间阻塞当前工作流，也更适合批量任务。

### Comet 视频

用于调用视频生成或视频编辑 API。

可能支持的模式包括：

- 文生。
- 图生。
- 首尾帧。
- 多图参考。
- 全能参考。
- 视频编辑。
- 运动控制。

不同渠道、模型支持的模式不同。插件会根据渠道和模型尽量显示合适的参数。

### Comet 异步视频提交 / Comet 异步视频收取

视频生成通常耗时更长，建议优先使用异步节点。

基本流程：

1. 用异步视频提交节点提交任务。
2. 保存任务 ID。
3. 等待平台生成完成。
4. 用异步视频收取节点获取结果。

### Comet 音乐

用于调用音乐生成 API。

当前主要用于 RunningHub Suno 系列模型。具体可用模型、时长、歌词等参数以渠道实际支持为准。

### Comet 文本

用于调用 LLM 文本模型。

适合：

- 生成提示词。
- 翻译文本。
- 解释报错。
- 代码辅助。
- 工作流说明生成。

文本模型接口格式通常包括 OpenAI 兼容、Gemini 原生、Claude Messages 等。

### Comet 图像卡片

用于在画布中保存和预览图像结果。

它可以把生成结果变成一个稳定的资源边界，后续节点复用这个图片时，不一定需要重新触发上游生成链。

### Comet 视频卡片

用于保存和预览视频结果。

适合把远端生成的视频结果保存在 ComfyUI 输出目录，并在工作流中继续传递。

### Comet 音频卡片

用于保存音频结果。

适合音乐生成、音频输出、音频资源整理等场景。

### Comet 文本卡片 / Comet 批量文本卡片 / Comet 文本追加

用于保存、传递和组合文本内容。

适合：

- 提示词管理。
- 批量文本输入。
- 多段文本拼接。
- 把 LLM 输出接入后续节点。

## 内置对话模块

插件内置 Comet 对话模块，可在 ComfyUI 内直接打开聊天窗口。

对话模块支持：

- 宽屏和半屏窗口模式。
- 多会话管理。
- 文本模型对话。
- Markdown 渲染。
- 代码块高亮。
- 表格显示。
- 回复复制。
- 重新生成。
- 图片、PDF、音频、视频、文本等附件输入。
- 图片粘贴到输入框。
- Gemini Image / gptimage 类图片模型生成。
- 生成图片复制、下载、重新生成、多版本翻页。

在 CometAPI 集成版中，对话模块可以读取 CometAPI 设置中心里已经配置好的文本模型渠道，减少重复配置。

## 私有渠道扩展

插件支持通过 `private_channels` 目录扩展私有渠道。

公开分发版不会内置具体私有渠道业务逻辑，只保留识别、加载和适配机制。

如需编写私有渠道，请阅读：

```text
private_channels/私有渠道适配指南.md
```

私有渠道通常由两个文件组成：

- `<channel_id>.json`
- `<channel_id>.py`

注意：

- 不要把 API Key 写进私有渠道文件。
- API Key 应在插件设置页填写。
- 不要把客户私有渠道逻辑提交到公开仓库。
- 修改私有渠道 Python 文件后通常需要重启 ComfyUI。

## 输出目录

插件生成的图片、视频、音频等资源会保存到 ComfyUI 的输出目录下。

常见目录包括：

```text
ComfyUI/output/cometapi_assets
ComfyUI/output/comet_chat
```

实际目录可能会随着节点类型和资源类型有所不同。

## 数据和配置

插件会在本地保存用户配置和对话数据。请不要把本地数据目录提交到公开仓库。

可能包含：

- API URL。
- API Key。
- 模型配置。
- 对话历史。
- 私有渠道配置。

这些内容属于用户本地数据，应妥善保管。

## 常见问题

### 这个插件自带 API Key 吗？

不自带。你需要使用自己的 API 渠道和 API Key。

### 这个插件绑定某个商业渠道吗？

不绑定。你可以使用自己想用的 API 渠道。不同渠道的模型、价格、稳定性、参数支持都以渠道实际情况为准。

### 为什么某个模型参数不生效？

不同渠道和模型支持的参数不同。比如比例、分辨率、参考图数量、视频时长、编辑模式等，不一定每个平台都支持。

### 为什么有些附件不能用？

不同文本接口对附件支持不同。OpenAI 兼容、Gemini 原生、Claude Messages 对图片、PDF、音频、视频的支持范围并不完全一致。

### 为什么图片或视频任务很慢？

远端生成任务耗时取决于平台排队、模型速度、分辨率、并发数量和网络情况。长任务建议使用异步节点。
