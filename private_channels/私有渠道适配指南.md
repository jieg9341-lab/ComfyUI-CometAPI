# 私有渠道 AI 适配指南

这份指南是给插件用户使用的。用户可以把本文和目标平台的 API 文档一起交给 AI，让 AI 生成匹配插件的私有渠道文件。

用户不需要了解插件前端实现，也不需要参考主程序源码。AI 只需要按本文的文件结构、JSON schema、适配器函数签名和返回格式来写代码。

最终产物只能是私有渠道文件：

- `<channel_id>.json`
- `<channel_id>.py`

不要要求用户修改 `nodes.py`、`web/*.js` 或插件分发版源码。

## 给 AI 的标准提示词

用户可以直接复制下面这段给 AI，然后把目标平台 API 文档贴在后面：

```text
你要帮我为 ComfyUI-CometAPI 插件编写一个私有渠道。

我会提供：
1. 私有渠道 AI 适配指南
2. 目标平台 API 文档
3. 我希望接入的模型列表和显示名称

请你只输出两个文件的完整内容：
1. <channel_id>.json
2. <channel_id>.py

要求：
- 不要修改插件源码、前端源码或 nodes.py。
- 不要假设我了解插件前端实现。
- JSON 必须是 UTF-8，不能出现乱码或 ????。
- JSON 负责声明渠道、模型、通用 UI 字段和素材数量限制。
- Python 适配器负责平台鉴权、上传、提交任务、轮询、下载结果和错误处理。
- 视频和音乐异步任务必须轮询到成功或失败。
- 提交任务接口不要盲目自动重试，避免重复提交和重复扣费。
- 错误信息要适合最终用户阅读，不要泄露 API Key、Bearer Token 或长签名 URL。
- 代码应尽量只依赖 Python 标准库、requests、Pillow；如需额外依赖必须说明。
- 不要发明 JSON 字段。模型项里只有 `family`、`api_format`、`interface_mode`、`capabilities`、`ui`、`runtime`、`adapter`、`endpoint` 等字段会被插件保留给适配器使用。
- 如果平台有“提交任务返回 task_id + 查询任务状态”接口，再写 `submit_*_task/query_*_task`；如果没有，只写 `generate_image/generate_video`。

请先根据 API 文档填写平台能力提取表，说明选择普通生成、函数内轮询、本地包装异步还是真异步，然后再给出最终两个文件。
```

## 用户需要提供给 AI 的信息

至少提供：

- 渠道 ID，例如 `example_api`。
- 渠道显示名，例如 `Example API`。
- 平台 Base URL。
- 平台 API 文档全文或关键接口页面。
- 要接入哪些模型，以及每个模型属于 `llm`、`image`、`video` 还是 `music`。
- 每个模型希望在节点横条显示哪些选项。
- API Key 统一让用户在插件设置页填写。

如果平台文档里没有写清楚上传、轮询或结果字段，要先让 AI 列出疑问，或者用平台 Playground/cURL 跑通后再写适配器。

## AI 输出格式要求

让 AI 最终按这个格式输出，方便用户直接落地成文件：

````text
文件：private_channels/<channel_id>.json
```json
{ ...完整 JSON... }
```

文件：private_channels/<channel_id>.py
```python
# ...完整 Python 代码...
```
````

AI 不应该输出“请修改前端”“请修改 nodes.py”“需要新增节点类”之类的方案。私有渠道只能通过 JSON 和 Python 适配器完成。

如果 AI 发现目标平台能力超出通用控件范围，应该优先选择以下做法：

- 在适配器里给特殊参数写默认值。
- 把不同能力拆成多个模型，例如 `model-fast`、`model-pro`。
- 把不同模式放进 `mode`，例如 `文生`、`图生`、`视频参考`。
- 只有平台文档不足时才向用户提问。

私有渠道由两个文件组成，放在 `private_channels` 目录：

- `<channel_id>.json`：渠道信息、模型列表、前端通用控件声明。
- `<channel_id>.py`：后端适配器，负责请求平台 API、上传素材、轮询任务、下载结果。

`channel_id` 建议只使用小写字母、数字和下划线。

## 适配边界

分发版只负责：

- 扫描 `private_channels/*.json`。
- 在设置页显示私有渠道和模型。
- 根据 JSON 里的 `ui` 渲染通用控件。
- 在运行节点时调用 `<channel_id>.py` 里的适配函数。

私有适配器负责：

- 平台认证和请求头。
- 模型名映射。
- 图片、视频、音频上传。
- 同步或异步任务提交。
- 任务轮询。
- 结果 URL 提取。
- 下载结果并转换成插件需要的返回值。
- 平台错误信息整理。

如果某个平台有特殊参数，但通用控件不支持，优先在适配器里写默认逻辑；如果确实需要用户选择，就拆成多个模型或多个 `mode`。

## 第一步：整理平台 API 文档

适配前先从平台文档里确认这些信息：

- Base URL，例如 `https://api.example.com`。
- 鉴权方式，通常是 `Authorization: Bearer <key>`。
- 支持的模型 ID、展示名、分类：文本、图片、视频、音乐。
- 每个模型支持的模式：文生、图生、首尾帧、多参考、视频参考等。
- 每个模型支持的参数：比例、尺寸、清晰度、秒数、分辨率、质量、风格等。
- 素材输入限制：参考图数量、参考视频数量、音频数量、文件大小、格式。
- 是否需要先上传文件，上传接口返回什么 URL。
- 生成接口是同步返回结果，还是返回任务 ID。
- 查询任务接口的状态字段和结果字段。
- 失败响应里的错误字段。

建议先用 cURL 或 Python requests 跑通一个最小请求，再写适配器。

## 平台能力提取表

AI 写代码前，先从平台 API 文档整理出这张表。表里任何关键项为空，都不要硬猜；先向用户确认或让用户提供接口响应样例。

| 项目 | 需要确认的内容 |
| --- | --- |
| 鉴权 | Header 名称、Token 格式、是否需要签名、时间戳或 nonce。 |
| Base URL | 只写到平台根地址或业务版本前缀，不要把单个 endpoint 写成 Base URL。 |
| 模型分类 | 每个模型属于 `llm`、`image`、`video`、`music` 哪一类。 |
| 用户控件 | 每个模型需要显示哪些固定字段：比例、尺寸、质量、模式、秒数、分辨率、音乐模式等。 |
| 素材输入 | 每个模式最多收几张图、几段视频、几段音频。 |
| 上传方式 | 平台是否接收 base64、公网 URL、本地文件上传、预签名上传或文件 ID。 |
| 生成接口 | 请求方法、路径、payload 字段、成功响应样例、失败响应样例。 |
| 任务 ID | 生成接口是否返回远端 `task_id`、`id` 或类似字段。 |
| 查询接口 | 是否有独立查询接口，状态字段路径、结果字段路径、失败原因字段路径。 |
| 结果格式 | 图片是 URL、base64 还是二进制；视频/音频是 URL、文件 ID 还是二进制。 |
| 幂等性 | 提交接口是否支持幂等键；不支持时不要自动重试提交。 |

## 适配决策树

按下面规则决定该写哪些函数：

```text
文本模型？
-> 优先写 generate_llm(context)
-> 如果是 OpenAI/Gemini/Claude 兼容协议，也可以只写 JSON 的 api_format，让插件走兼容逻辑。

图片模型？
-> 平台同步返回图片/base64/URL：写 generate_image(context)，返回 PIL.Image 列表。
-> 平台有 task_id + 查询接口：可以额外写 submit_image_task/query_image_task，异步图片节点会优先走真异步。
-> 平台没有 task_id：不要写 submit/query，异步图片节点会自动把 generate_image 包装成本地异步任务。

视频模型？
-> 平台生成接口内部需要轮询，但没有必要暴露给收取节点：写 generate_video(context)，函数内轮询到完成后返回本地 path。
-> 平台有 task_id + 查询接口，并希望提交后以后再收：写 submit_video_task/query_video_task。
-> 平台没有可查询 task_id：只写 generate_video，异步视频节点会自动包装成本地异步任务。

音乐模型？
-> 写 generate_music(context)，函数内轮询到完成后返回 AUDIO。
```

不要为了某个平台的特殊参数去改 `nodes.py` 或前端。通用控件不够时，优先把不同能力拆成多个模型，或者在适配器里写默认值。

## JSON 文件

最小结构：

```json
{
  "enabled": true,
  "id": "example",
  "name": "Example API",
  "api_url": "https://api.example.com",
  "adapter": "example",
  "models": []
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `enabled` | 否 | `false` 时禁用该渠道。 |
| `id` | 是 | 渠道 ID，必须和文件名、设置里的渠道键保持一致。 |
| `name` | 是 | 设置页显示名。 |
| `api_url` | 是 | 平台 Base URL。适配器可从 `context["api_url"]` 读取。 |
| `adapter` | 是 | 对应 `<adapter>.py`，通常和 `id` 一致。 |
| `models` | 是 | 模型列表。 |

私有渠道的 API Key 不写进 JSON，统一由插件设置页提供，运行时注入到 `context["api_key"]`。

模型字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `model` | 是 | 真实模型 ID，适配器收到的 `context["model"]`。 |
| `alias` | 否 | UI 展示名。 |
| `category` | 是 | `llm`、`image`、`video`、`music`。 |
| `family` | 否 | 视频模型分组或适配器内部分类，例如 `veo`、`kling`、`suno`。 |
| `api_format` | 否 | 文本/图片兼容格式提示，例如 `openai`、`gemini`、`gpt_image`、`gemini_image`。 |
| `ui` | 否 | 通用控件和素材限制。 |
| `runtime` / `adapter` | 否 | 私有适配器自用配置，会原样传入 `context["model_settings"]`。 |

模型项字段白名单：

| 字段 | 是否会保留到 `context["model_settings"]` | 用途 |
| --- | --- | --- |
| `model` | 是 | 真实模型 ID。 |
| `alias` | 是 | UI 展示名。 |
| `category` | 是 | 模型分类。 |
| `family` | 是 | 前端推断默认横条，适配器也可读取。 |
| `api_key` | 是 | 单模型专用 Key，不建议常规使用。 |
| `api_format` | 是 | 文本/图片兼容格式。 |
| `interface_mode` | 是 | 图片兼容接口模式。 |
| `capabilities` | 是 | 旧写法，作用接近 `ui`。 |
| `ui` | 是 | 控件选项和素材限制。 |
| `runtime` | 是 | 推荐放平台特殊参数、真实 endpoint、内部映射等适配器自用配置。 |
| `adapter` | 是 | 模型级适配器自用配置。 |
| `endpoint` | 是 | 模型级 endpoint 配置。 |

不要在模型项里随意添加 `foo`、`extra_params`、`payload` 这类自定义字段；分发版规范化 JSON 时不会保留它们。需要给适配器的特殊配置，统一放进 `runtime`、`adapter` 或 `endpoint`。

## UI Schema

### 通用规则

每个字段都可以写：

```json
{
  "values": ["选项1", "选项2"],
  "default": "选项1",
  "values_by_mode": {
    "某个模式": ["选项2"]
  },
  "values_by_duration": {
    "10": ["720p"]
  }
}
```

`values_by_mode` 用于“某个模式下只允许部分选项”。`default` 必须存在于 `values` 里，否则会回退到第一个选项。

插件实际支持两种等价写法，推荐使用 `ui.fields`：

```json
"ui": {
  "fields": {
    "aspect_ratio": { "values": ["16:9", "9:16"], "default": "16:9" }
  }
}
```

兼容的扁平写法：

```json
"ui": {
  "aspect_ratios": ["16:9", "9:16"],
  "default_aspect_ratio": "16:9"
}
```

字段值读取规则：

| 规则 | 说明 |
| --- | --- |
| `values` / `choices` | 当前字段的基础选项。 |
| `default` | 默认值，必须在当前可选项中。 |
| `values_by_mode` / `choices_by_mode` | 按 `mode` 过滤选项。 |
| `values_by_duration` / `choices_by_duration` | 按 `duration` 过滤选项。 |
| 没有可选项 | 对私有视频会隐藏该字段；对私有图片会回退到 family 或默认选项。 |

当前前端只读取固定字段名，不支持自定义新控件：

| 分类 | 支持的 `ui.fields` 字段 |
| --- | --- |
| 图片 | `aspect_ratio`、`image_size`、`quality` |
| 视频 | `mode`、`aspect_ratio`、`duration`、`resolution`、`size` |
| 音乐 | `music_mode`、`music_submode`、`vocal_gender` |

如果平台有 `style`、`seed`、`negative_prompt`、`camera` 之类通用控件没有的参数，不要写成新 UI 字段。优先在 `runtime` 中写默认值，或拆成多个模型让用户选择。

所有 JSON 文件必须保存为 UTF-8。不要出现 `????` 或乱码，否则前端横条会原样显示乱码。

### 图片模型

常用字段：

- `aspect_ratio`：比例，例如 `auto`、`1:1`、`16:9`、`9:16`。
- `image_size`：尺寸或清晰度，例如 `1K`、`2K`、`4K`。
- `quality`：质量，例如 `low`、`medium`、`high`。

示例：

```json
{
  "model": "example-image",
  "alias": "Example Image",
  "category": "image",
  "family": "example",
  "ui": {
    "fields": {
      "aspect_ratio": {
        "values": ["auto", "1:1", "16:9", "9:16"],
        "default": "auto"
      },
      "image_size": {
        "values": ["1K", "2K", "4K"],
        "default": "2K"
      },
      "quality": {
        "values": ["low", "medium", "high"],
        "default": "medium"
      }
    },
    "media_limits": {
      "image": 4,
      "video": 0,
      "audio": 0
    }
  }
}
```

### 视频模型

常用字段：

- `mode`：模式，例如 `文生`、`图生`、`首尾帧`、`多参`、`视频参考`。
- `aspect_ratio`：比例，例如 `16:9`、`9:16`。
- `duration`：秒数，例如 `5`、`10`。
- `resolution`：分辨率，例如 `720p`、`1080p`、`4K`。
- `size`：部分平台用 `size` 表示比例或输出规格。

素材限制：

```json
"media_limits": {
  "image": 0,
  "video": 0,
  "audio": 0,
  "by_mode": {
    "图生": { "image": 1, "video": 0, "audio": 0 },
    "首尾帧": { "image": 2, "video": 0, "audio": 0 },
    "视频参考": { "image": 0, "video": 1, "audio": 0 }
  }
}
```

`media_limits` 会同时影响前端素材输入口和后端运行校验：

| 字段 | 说明 |
| --- | --- |
| `image` | 当前默认模式最多接收几张参考图。 |
| `video` | 当前默认模式最多接收几段参考视频。 |
| `audio` | 当前默认模式最多接收几段参考音频。 |
| `by_mode` | 按 `mode` 覆盖素材数量，模式名不区分大小写但建议和 UI 选项完全一致。 |

图片模型也建议写 `media_limits`，否则前端和后端会使用插件默认参考图数量。视频模型如果完全不写 `media_limits`，为了兼容旧私有视频配置，后端会默认允许图片素材；新适配不要依赖这个兜底，必须显式写清楚。

如果某个模式不接收任何素材，写 `{ "image": 0, "video": 0, "audio": 0 }`。如果平台文档只支持公网 URL 而插件素材输入是本地图片/视频，适配器必须负责上传或转换，不能只把本地路径当公网 URL 传给平台。

视频节点会根据已连接素材做一次局部自动切模式：当前模式是 `文生`，并且用户给这个视频节点接入图片/视频/音频素材时，前端会只修改这个节点自己的 `mode` 控件，自动切到能接收这些素材的非文生模式。这个逻辑不刷新整张画布，不会执行其他节点，也不会替私有渠道改业务逻辑。

私有渠道要让这个自动切换可靠生效，必须同时声明 `ui.fields.mode.values` 和 `ui.media_limits.by_mode`。例如 `文生` 不接素材就显式写 `{ "image": 0, "video": 0, "audio": 0 }`，`图生` 写允许图片数量，`视频参考` 写允许视频数量。不要只写模式名让插件猜素材能力；如果没有 `by_mode`，插件会按旧私有视频配置做兼容，不适合作为新适配的依据。

如果用户已经手动把模式切回 `文生`，即使素材还连着，插件也不会再次强制切回图生或视频参考。适配器仍然要在 `generate_video` / `submit_video_task` 里按平台规则校验素材数量，不能依赖前端模式切换代替后端校验。

完整示例：

```json
{
  "model": "example-video",
  "alias": "Example Video",
  "category": "video",
  "family": "example",
  "ui": {
    "fields": {
      "mode": {
        "values": ["文生", "图生", "首尾帧", "视频参考"],
        "default": "文生"
      },
      "aspect_ratio": {
        "values": ["16:9", "9:16"],
        "default": "16:9"
      },
      "duration": {
        "values": ["5", "10"],
        "default": "5"
      },
      "resolution": {
        "values": ["720p", "1080p"],
        "default": "720p"
      }
    },
    "media_limits": {
      "image": 0,
      "video": 0,
      "audio": 0,
      "by_mode": {
        "图生": { "image": 1, "video": 0, "audio": 0 },
        "首尾帧": { "image": 2, "video": 0, "audio": 0 },
        "视频参考": { "image": 0, "video": 1, "audio": 0 }
      }
    }
  }
}
```

通用私有渠道不要求前端按模式隐藏字段。如果某个平台在某个模式下不需要 `duration`、`resolution` 之类的参数，适配器在组装 payload 时忽略它即可。若必须让用户完全看不到某个参数，建议拆成单独模型或单独私有渠道版本。

### 音乐模型

常用字段：

- `music_mode`：建议 `生成歌曲`、`生成歌词`。
- `music_submode`：建议 `灵感模式`、`自定义歌词`、`纯音乐`、`生成歌词`。
- `vocal_gender`：建议 `自动`、`男声`、`女声`。

示例：

```json
{
  "model": "chirp-fenix",
  "alias": "Suno v5.5",
  "category": "music",
  "family": "suno",
  "ui": {
    "fields": {
      "music_mode": {
        "values": ["生成歌曲", "生成歌词"],
        "default": "生成歌曲"
      },
      "music_submode": {
        "values": ["灵感模式", "自定义歌词", "纯音乐", "生成歌词"],
        "values_by_mode": {
          "生成歌曲": ["灵感模式", "自定义歌词", "纯音乐"],
          "生成歌词": ["生成歌词"]
        },
        "default": "灵感模式"
      },
      "vocal_gender": {
        "values": ["自动", "男声", "女声"],
        "default": "自动"
      }
    }
  }
}
```

## 适配器文件

适配器文件名必须与 JSON 的 `adapter` 一致，例如：

- `example.json`
- `example.py`

适配器可以实现以下函数中的一个或多个：

```python
def generate_llm(context): ...
def generate_image(context): ...
def generate_video(context): ...
def generate_music(context): ...
```

未实现的分类不会影响其他分类。

文本和图片如果是 OpenAI/Gemini 兼容协议，可以不写对应函数，让插件走兼容逻辑。视频和音乐通常必须写适配器。

函数选择表：

| 场景 | 必写函数 | 可选函数 | 插件行为 |
| --- | --- | --- | --- |
| 文本普通节点 | `generate_llm` | 无 | 调用后输出文本。兼容协议可不写。 |
| 图片普通节点 | `generate_image` | 无 | 调用后输出 `PIL.Image` 列表。兼容图片协议可不写。 |
| 图片异步提交/收取 | `generate_image` | `submit_image_task` + `query_image_task` | 有 submit/query 就走远端真异步；否则后台包装 `generate_image`。 |
| 批量图片异步提交/收取 | `generate_image` | `submit_image_task` + `query_image_task` | 和图片异步共用收取节点；无远端 ID 时逐条后台包装。 |
| 视频普通节点 | `generate_video` | 无 | 函数内完成提交、轮询、下载，然后返回本地视频 path。 |
| 视频异步提交/收取 | `generate_video` | `submit_video_task` + `query_video_task` | 有 submit/query 就走远端真异步；否则后台包装 `generate_video`。 |
| 音乐普通节点 | `generate_music` | 无 | 函数内完成提交、轮询、下载，然后返回 ComfyUI AUDIO。 |

旧私有渠道只要普通 `generate_image` / `generate_video` 能跑，就能被新的图片/视频异步节点使用。区别只是：没有远端 `submit/query` 时，异步任务运行在当前 ComfyUI 进程的后台线程里。

### 异步节点适配

Comet 异步节点由分发版代码统一管理任务队列、本地 Comet 任务 ID、状态面板、收取节点和结果缓存。私有渠道适配器只需要声明平台自己的能力。

如果平台生成接口会返回远端任务 ID，并且有查询任务接口，建议实现真异步函数：

```python
def submit_image_task(context): ...
def query_image_task(context): ...
def submit_video_task(context): ...
def query_video_task(context): ...
```

真异步函数是可选的，不影响普通 `generate_image` / `generate_video` 节点。

如果平台没有远端任务 ID，或者文档只提供同步生成接口，不要为了异步节点修改分发版源码。只实现普通同步函数即可：

```python
def generate_image(context): ...
def generate_video(context): ...
```

异步提交节点会自动把这类同步生成函数包装成本地 Comet 任务：提交节点先写本地任务记录并返回 Comet 任务 ID，后台线程调用 `generate_image` / `generate_video`，完成后把结果写入本地缓存，收取节点再读取结果。

两种模式的分工：

| 模式 | 私有适配器负责 | 分发版负责 |
| --- | --- | --- |
| 真异步 | 提交远端任务、查询远端状态、提取结果 URL 或本地路径 | 保存任务 ID、刷新状态、收取结果、显示队列 |
| 本地包装异步 | 同步生成图片或视频并返回结果 | 后台执行、生成本地 Comet 任务 ID、缓存结果、收取结果 |

注意：本地包装异步依赖当前 ComfyUI 进程里的后台线程。ComfyUI 重启后，已经写入缓存的结果仍可收取，正在运行但尚未完成的本地包装任务会丢失，并在查询时标记为失败。真异步任务只要远端任务 ID 仍有效，重启后仍可继续查询。

真异步返回格式：

```python
def submit_image_task(context):
    return {
        "task_id": "remote-task-id",
        "adapter_data": {}
    }

def query_image_task(context):
    return {
        "status": "succeeded",
        "image_urls": ["https://example.com/result.png"]
    }

def submit_video_task(context):
    return {
        "task_id": "remote-task-id",
        "adapter_data": {}
    }

def query_video_task(context):
    return {
        "status": "succeeded",
        "video_url": "https://example.com/result.mp4"
    }
```

`status` 可以返回平台原始状态，分发版会归一化常见值。建议成功返回 `succeeded`，运行中返回 `running`，失败返回 `failed`。失败时返回 `error` 或 `failure_reason`：

```python
return {"status": "failed", "failure_reason": "平台返回的失败原因"}
```

`query_*_task(context)` 会收到：

| 字段 | 说明 |
| --- | --- |
| `task_id` | `submit_*_task` 返回的远端任务 ID。 |
| `adapter_data` | `submit_*_task` 返回的完整 dict，可用来保存平台需要的额外查询参数。 |

视频真异步如果平台查询结果已经是本地文件，可以返回 `path` 或 `local_path`。如果返回的是公网 URL，写 `video_url` 即可，收取节点会下载并交给视频卡片继续保存。

### 适配器常用工具函数

建议 AI 在 `<channel_id>.py` 里自带这些小工具，再按目标平台响应结构调整字段路径。不要依赖未定义的 `extract_image_urls`、`extract_video_url`。

```python
from io import BytesIO
import base64
import time

import requests
from PIL import Image


def _headers(api_key):
    return {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "Accept": "application/json",
    }


def _request_json(method, url, api_key, *, json=None, timeout=(15, 90), CometAPIError=RuntimeError):
    response = requests.request(method, url, headers=_headers(api_key), json=json, timeout=timeout)
    if response.status_code >= 400:
        raise CometAPIError(f"平台请求失败：HTTP {response.status_code} {response.text[:500]}")
    try:
        return response.json()
    except Exception as exc:
        raise CometAPIError(f"平台没有返回 JSON：{response.text[:300]}") from exc


def _pick(data, path, default=None):
    value = data
    for key in str(path).split("."):
        if isinstance(value, dict):
            value = value.get(key)
        elif isinstance(value, list) and key.isdigit() and int(key) < len(value):
            value = value[int(key)]
        else:
            return default
    return value if value is not None else default


def _first_url(data, paths):
    for path in paths:
        value = _pick(data, path)
        if isinstance(value, str) and value.startswith(("http://", "https://")):
            return value
        if isinstance(value, list):
            for item in value:
                if isinstance(item, str) and item.startswith(("http://", "https://")):
                    return item
                if isinstance(item, dict):
                    url = item.get("url") or item.get("image_url") or item.get("video_url")
                    if isinstance(url, str) and url.startswith(("http://", "https://")):
                        return url
    return ""


def _image_from_url(url, CometAPIError=RuntimeError):
    response = requests.get(url, timeout=(15, 300))
    if response.status_code >= 400:
        raise CometAPIError(f"图片下载失败：HTTP {response.status_code}")
    return Image.open(BytesIO(response.content)).convert("RGB")


def _image_from_base64(value, CometAPIError=RuntimeError):
    raw = str(value or "")
    if "," in raw and raw.startswith("data:"):
        raw = raw.split(",", 1)[1]
    try:
        return Image.open(BytesIO(base64.b64decode(raw))).convert("RGB")
    except Exception as exc:
        raise CometAPIError("图片 base64 解码失败。") from exc
```

### 最小完整示例：图片同步 + 视频函数内轮询

下面是一套完整骨架，适合大多数“图片接口直接返回结果、视频接口返回任务 ID 但由普通节点内部轮询”的平台。把 endpoint、payload 和结果字段路径按目标平台文档替换即可。

`example.py`：

```python
from io import BytesIO
import time

import requests
from PIL import Image


def _headers(api_key):
    return {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "Accept": "application/json",
    }


def _request_json(method, url, api_key, *, json=None, timeout=(15, 90), CometAPIError=RuntimeError):
    response = requests.request(method, url, headers=_headers(api_key), json=json, timeout=timeout)
    if response.status_code >= 400:
        raise CometAPIError(f"平台请求失败：HTTP {response.status_code} {response.text[:500]}")
    return response.json()


def _first_url(data):
    # 按目标平台实际响应调整这些路径。
    if not isinstance(data, dict):
        return ""
    candidates = [
        data.get("url"),
        ((data.get("data") or {}).get("url") if isinstance(data.get("data"), dict) else ""),
    ]
    if isinstance(data.get("data"), list):
        for item in data["data"]:
            if isinstance(item, dict):
                candidates.append(item.get("url") or item.get("image_url") or item.get("video_url"))
    for value in candidates:
        if isinstance(value, str) and value.startswith(("http://", "https://")):
            return value
    return ""


def _download_image(url, CometAPIError):
    response = requests.get(url, timeout=(15, 300))
    if response.status_code >= 400:
        raise CometAPIError(f"图片下载失败：HTTP {response.status_code}")
    return Image.open(BytesIO(response.content)).convert("RGB")


def _upload_image_if_needed(context, image):
    # 如果平台支持 base64，可以在这里返回 base64。
    # 如果平台有上传接口，在这里上传并返回公网 URL 或 file_id。
    # 如果平台文档不需要参考图，直接返回 ""。
    return ""


def generate_image(context):
    CometAPIError = context["helpers"]["CometAPIError"]

    api_key = context["api_key"]
    base_url = context["api_url"].rstrip("/")
    runtime = context["model_settings"].get("runtime") or {}
    endpoint = runtime.get("endpoint") or "/v1/images/generations"

    prompt = context["prompt"].strip()
    if not prompt:
        raise CometAPIError("请填写提示词。")

    image_refs = []
    for image in context["pil_images"]:
        uploaded = _upload_image_if_needed(context, image)
        if uploaded:
            image_refs.append(uploaded)

    payload = {
        "model": context["model"],
        "prompt": prompt,
        "aspect_ratio": context["aspect_ratio"],
        "size": context["image_size"],
    }
    if image_refs:
        payload["image_urls"] = image_refs

    data = _request_json("POST", f"{base_url}{endpoint}", api_key, json=payload, timeout=(15, 300), CometAPIError=CometAPIError)
    image_url = _first_url(data)
    if not image_url:
        raise CometAPIError(f"图片接口没有返回图片 URL：{str(data)[:300]}")
    return {"images": [_download_image(image_url, CometAPIError)], "errors": []}


def generate_video(context):
    CometAPIError = context["helpers"]["CometAPIError"]
    download_video_asset = context["helpers"]["download_video_asset"]

    api_key = context["api_key"]
    base_url = context["api_url"].rstrip("/")
    runtime = context["model_settings"].get("runtime") or {}
    submit_endpoint = runtime.get("submit_endpoint") or "/v1/videos/generations"
    query_endpoint = runtime.get("query_endpoint") or "/v1/videos/tasks/{task_id}"

    prompt = context["prompt"].strip()
    if not prompt:
        raise CometAPIError("请填写提示词。")

    payload = {
        "model": context["model"],
        "prompt": prompt,
        "mode": context["mode"],
        "aspect_ratio": context["aspect_ratio"],
        "duration": int(context["duration"] or 5),
        "resolution": context["resolution"],
    }
    if context["mode"] == "图生":
        if not context["pil_images"]:
            raise CometAPIError("图生模式需要 1 张参考图。")
        uploaded = _upload_image_if_needed(context, context["pil_images"][0])
        if uploaded:
            payload["image_url"] = uploaded

    submit = _request_json("POST", f"{base_url}{submit_endpoint}", api_key, json=payload, timeout=(15, 90), CometAPIError=CometAPIError)
    task_id = submit.get("task_id") or submit.get("id") or (submit.get("data") or {}).get("task_id")
    if not task_id:
        raise CometAPIError(f"视频提交没有返回任务 ID：{str(submit)[:300]}")

    started = time.time()
    while time.time() - started < 1800:
        url = f"{base_url}{query_endpoint.format(task_id=task_id)}"
        data = _request_json("GET", url, api_key, timeout=(15, 30), CometAPIError=CometAPIError)
        status = str(data.get("status") or (data.get("data") or {}).get("status") or "").lower()
        video_url = _first_url(data)
        if status in {"success", "succeeded", "completed", "complete", "finished", "done"} or video_url:
            if not video_url:
                raise CometAPIError(f"视频任务完成但没有返回视频 URL：{str(data)[:300]}")
            local_path = download_video_asset(video_url, prefix="CometAPIExample")
            return {"path": local_path, "video_url": video_url, "task_id": str(task_id)}
        if status in {"failed", "failure", "fail", "error", "cancelled", "canceled", "expired"}:
            reason = data.get("error") or data.get("message") or data.get("failure_reason") or str(data)[:300]
            raise CometAPIError(f"视频任务失败：{reason}")
        time.sleep(5)

    raise CometAPIError(f"视频任务等待超时：{task_id}")
```

### 真异步图片模板

平台如果提交接口返回任务 ID，图片异步节点会优先使用这组函数：

```python
import requests


def _headers(api_key):
    return {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "Accept": "application/json",
    }


def submit_image_task(context):
    CometAPIError = context["helpers"]["CometAPIError"]

    api_key = context["api_key"]
    base_url = context["api_url"].rstrip("/")
    payload = {
        "model": context["model"],
        "prompt": context["prompt"],
        "aspect_ratio": context["aspect_ratio"],
        "image_size": context["image_size"],
    }

    response = requests.post(
        f"{base_url}/v1/images/tasks",
        headers=_headers(api_key),
        json=payload,
        timeout=(15, 90),
    )
    if response.status_code >= 400:
        raise CometAPIError(response.text[:500])
    data = response.json()
    task_id = data.get("task_id") or data.get("id") or data.get("data", {}).get("task_id")
    if not task_id:
        raise CometAPIError(f"图片提交没有返回任务 ID：{str(data)[:300]}")
    return {"task_id": str(task_id), "adapter_data": data}


def query_image_task(context):
    api_key = context["api_key"]
    base_url = context["api_url"].rstrip("/")
    task_id = context["task_id"]

    response = requests.get(
        f"{base_url}/v1/images/tasks/{task_id}",
        headers=_headers(api_key),
        timeout=(15, 30),
    )
    if response.status_code >= 400:
        return {"status": "failed", "failure_reason": response.text[:500]}
    data = response.json()
    status = str(data.get("status") or data.get("data", {}).get("status") or "running").lower()
    image_urls = extract_image_urls(data)
    return {"status": status, "image_urls": image_urls}
```

上面的 `extract_image_urls(data)` 是示例占位函数，需要按平台实际响应结构自己实现。只有在状态已经成功时才需要返回结果 URL；运行中可以返回空列表。

如果平台没有任务 ID，不要写 `submit_image_task/query_image_task`，只写下面的普通 `generate_image` 模板即可，异步节点会自动走本地包装异步。

### 真异步视频模板

平台如果提交接口返回任务 ID，视频异步节点会优先使用这组函数：

```python
import requests


def _headers(api_key):
    return {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "Accept": "application/json",
    }


def submit_video_task(context):
    CometAPIError = context["helpers"]["CometAPIError"]

    api_key = context["api_key"]
    base_url = context["api_url"].rstrip("/")
    payload = {
        "model": context["model"],
        "prompt": context["prompt"],
        "mode": context["mode"],
        "aspect_ratio": context["aspect_ratio"],

    if not task_id:

    base_url = context["api_url"].rstrip("/")
    task_id = context["task_id"]

    response = requests.get(

```

上面的 `extract_video_url(data)` 是示例占位函数，需要按平台实际响应结构自己实现。只有在状态已经成功时才需要返回视频 URL；运行中可以返回空字符串或不返回该字段。

如果平台没有任务 ID，不要写 `submit_video_task/query_video_task`，只写普通 `generate_video`。异步节点会自动在本地后台执行 `generate_video`，完成后由收取节点读取本地结果。

### 普通视频模板（函数内轮询）

这个模板用于只实现 `generate_video` 的普通视频节点。平台虽然可能在接口内部有任务 ID，但这里的任务提交和轮询都发生在同一次节点执行里；如果想让 Comet 异步提交节点先提交、后续由收取节点查询，请改用上面的 `submit_video_task/query_video_task` 真异步模板。

```python
import time
import requests


def _headers(api_key):
    return {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "Accept": "application/json",
    }


def _post_json(url, api_key, payload, timeout=90):
    response = requests.post(url, headers=_headers(api_key), json=payload, timeout=(15, timeout))
    if response.status_code >= 400:
        raise RuntimeError(response.text[:500])
    return response.json()


def _get_json(url, api_key, timeout=30):
    response = requests.get(url, headers=_headers(api_key), timeout=(15, timeout))
    if response.status_code >= 400:
        raise RuntimeError(response.text[:500])
    return response.json()


def generate_video(context):
    CometAPIError = context["helpers"]["CometAPIError"]
    download_video_asset = context["helpers"]["download_video_asset"]

    api_key = context["api_key"]
    base_url = context["api_url"].rstrip("/")
    model = context["model"]
    prompt = context["prompt"].strip()
    mode = context["mode"]

    if not prompt:
        raise CometAPIError("请填写提示词。")

    payload = {
        "model": model,
        "prompt": prompt,
        "aspect_ratio": context["aspect_ratio"],
        "duration": int(context["duration"] or 5),
        "resolution": context["resolution"],
    }

    if mode == "图生":
        images = context["pil_images"]
        if not images:
            raise CometAPIError("图生模式需要 1 张参考图。")
        # 根据平台文档实现上传图片，得到 image_url
        payload["image_url"] = upload_image_to_platform(base_url, api_key, images[0])

    try:
        submit = _post_json(f"{base_url}/v1/videos/generations", api_key, payload)
        task_id = submit["data"]["task_id"]
    except Exception as exc:
        raise CometAPIError(f"视频任务提交失败：{exc}") from exc

    started = time.time()
    timeout = 1800
    while time.time() - started < timeout:
        data = _get_json(f"{base_url}/v1/tasks/{task_id}", api_key)
        status = str(data.get("status") or data.get("data", {}).get("status") or "").lower()
        if status in {"completed", "success", "succeeded"}:
            result_url = extract_video_url(data)
            if not result_url:
                raise CometAPIError(f"任务完成但没有返回视频 URL：{str(data)[:300]}")
            local_path = download_video_asset(result_url, prefix="CometAPIExample")
            return {"path": local_path, "video_url": result_url, "task_id": task_id}
        if status in {"failed", "error", "cancelled", "canceled"}:
            raise CometAPIError(f"视频任务失败：{str(data)[:500]}")
        time.sleep(5)

    raise CometAPIError(f"视频任务等待超时：{task_id}")
```

上面的 `upload_image_to_platform` 和 `extract_video_url` 都是示例占位函数，交付前必须按目标平台文档实现，不能原样留下未定义函数。

提交请求如果已经成功到达平台，但响应中途断开，不要盲目自动重试“提交任务”接口，否则可能重复扣费。可以重试查询接口，提交接口是否重试必须根据平台是否支持幂等键决定。

    image.convert("RGB").save(buffer, format="PNG")
    return buffer.getvalue()

    api_key = context["api_key"]

    prompt = context["prompt"].strip()

    payload = {

        "aspect_ratio": context["aspect_ratio"],
        "image_urls": image_urls,
    }

    response = requests.post(

        headers={"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"},
        json=payload,
        timeout=(15, 300),
    )
    if response.status_code >= 400:
        raise CometAPIError(response.text[:500])

    result_urls = extract_image_urls(response.json())
    images = []
    for url in result_urls:

    return {"images": images, "errors": []}

## 处理素材输入

图片输入已经是 `PIL.Image`，直接上传即可。

视频和音频输入可能是以下形式：

- 已经是公网 URL 的字符串。
- 本地路径字符串。
- 对象里带 `url`、`video_url`、`audio_url`。
- 对象里带 `video_path`、`audio_path`、`path`。
- 对象带 `save_to(path)` 方法。

建议适配器写一个 `media_to_path_or_url(value)` 辅助函数：

```python
def media_url(value):
    if isinstance(value, str) and value.startswith(("http://", "https://")):
        return value
    if isinstance(value, dict):
        for key in ("url", "video_url", "audio_url", "src"):
            raw = str(value.get(key) or "")
            if raw.startswith(("http://", "https://")):
                return raw
    for attr in ("url", "video_url", "audio_url", "src"):
        raw = str(getattr(value, attr, "") or "")
        if raw.startswith(("http://", "https://")):
            return raw
    return ""
```

如果平台要求所有素材都先上传，就把本地路径或 `save_to` 生成的临时文件上传到平台图床，再把 URL 放进生成 payload。

这里说的“图床”优先指目标平台自己的上传接口、预签名上传、对象存储直传或平台文档明确支持的上传流程。不要依赖插件分发版里其他渠道借来的上传逻辑。

如果目标平台根本没有上传接口，只接受公网 URL，那就让用户自己提供已经公开可访问的 URL，或在用户自己控制的对象存储/CDN 上先传好文件，再把 URL 交给适配器。

兜底策略可以这样写给 AI：

- 目标平台文档里有上传接口，就优先用目标平台自己的上传接口。
- 目标平台文档里直接支持 `base64`，就直接走 `base64`，不必额外上传。
- 目标平台没有上传能力、又必须要公开 URL 时，才可以借用 RunningHub 的上传服务作为兜底，先传成公开 URL，再喂给目标平台。
- 如果连 RunningHub 兜底都不合适，就让用户自己提供公网可访问的 URL，或者先传到他们自己的对象存储/CDN。

对图片模型尤其如此。很多平台文档会直接给出 `image_url`、`image_urls`、`input_image`、`base64` 这类写法，AI 应该先按平台文档走，不要先假设一定要借图床。

## 模型映射建议

把“用户看到的模型”和“平台真实模型”分开：

- JSON 的 `alias` 写展示名。
- JSON 的 `model` 写适配器内部稳定 ID。
- 如果平台同一个模型因清晰度、模式不同要传不同 ID，在适配器里用 `model + mode + resolution` 做映射。

示例：

```python
def resolve_real_model(model, mode, resolution):
    if model == "veo3.1-fast" and mode == "多参":
        return "veo3.1-fast-components-4K"
    if model == "veo3.1-fast" and resolution == "4K":
        return "veo3.1-fast-4K"
    return model
```

## 错误处理

适配器应该抛出用户能看懂的错误：

```python
raise CometAPIError("图生模式需要 1 张参考图。")
raise CometAPIError(f"平台请求失败：HTTP {response.status_code} {message}")
```

不要把完整 API Key、签名 URL、Bearer Token 打印到控制台。需要记录错误时先脱敏：

```python
redact = context["helpers"]["redact_sensitive_text"]
print(redact(str(exc)))
```

轮询查询接口可以重试；提交生成任务接口不要随便重试。除非平台文档明确支持幂等键，否则网络中断时重复提交可能导致重复任务和重复扣费。

## 常见错误反例

下面这些写法不要交付：

| 错误写法 | 为什么错 | 正确做法 |
| --- | --- | --- |
| 让用户修改 `nodes.py` 或 `web/*.js` | 私有渠道只能通过 JSON + Python 适配器完成。 | 只交付 `<channel_id>.json` 和 `<channel_id>.py`。 |
| 在 JSON 模型里新增 `style`、`seed`、`payload` 等字段 | 分发版规范化时不会保留任意字段。 | 放到 `runtime`、`adapter` 或 `endpoint`。 |
| 在 `ui.fields` 里新增 `style` 控件 | 前端只读取固定字段名，不会渲染自定义控件。 | 拆成多个模型，或在适配器里写默认值。 |
| `generate_image` 返回 `["https://...png"]` | 图片节点需要 `PIL.Image` 列表。 | 下载 URL 并转成 `PIL.Image`。 |
| `generate_video` 只返回公网 URL | 视频节点需要本地 path 才能稳定输出。 | 用 `download_video_asset(video_url, prefix=...)` 下载后返回 `path`。 |
| 平台无远端任务 ID 却强行写 `submit_*_task` | 收取节点无法查询真实状态。 | 只写 `generate_image/generate_video`，让异步节点本地包装。 |
| 查询函数运行中返回 `failed` | 会让任务提前失败。 | 运行中返回 `running`、`pending`、`processing` 等。 |
| 失败时吞掉平台错误 | 用户无法判断是 Key、余额、素材还是模型问题。 | 返回或抛出用户可读失败原因，并脱敏。 |
| 把 API Key 写进 JSON 或代码 | 交付泄密。 | 让用户在插件设置页填写，适配器读 `context["api_key"]`。 |
| 对提交接口自动重试 | 可能重复生成、重复扣费。 | 只重试查询接口；提交重试必须有幂等键。 |
